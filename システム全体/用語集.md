# 用語集

本ドキュメントは、`遠隔お知らせ掲示板` プロジェクトで使う主要用語の定義をまとめたものです。

## システム構成

| 用語 | 定義 |
| --- | --- |
| 遠隔お知らせ掲示板 | 本プロジェクトで検討している、遠隔から自宅端末へお知らせを配信する仕組み全体。 |
| クラウド | Webサーバー、APIサーバー、配信ワーカー、データベースで構成されるサーバー側システム。 |
| ゲートウェイ | ユーザー自宅に配置され、クラウドと端末の間を中継する装置（`remoboard-gateway-api`）。 |
| 端末 | ゲートウェイ配下で、お知らせを実際に表示するデバイス。 |
| 模擬ゲートウェイ | 初期段階で実機の代わりに利用する、検証用のゲートウェイ実装。 |
| 模擬端末 | 初期段階で実機の代わりに利用する、検証用の端末実装。 |

## ロール・ユーザー

| 用語 | 定義 |
| --- | --- |
| admin管理者 | システム全体を管理する最上位ロール。システム設定・全体監視・権限管理を行う。 |
| 一般管理者 | `adminユーザー` を管理するロール。作成・停止・所属管理を行う。 |
| adminユーザー | 担当するゲートウェイ群と一般ユーザーを管理するロール。 |
| 一般ユーザー | 実際にお知らせを作成し、端末へ配信するロール。 |
| 所有関係 | ロールごとに管理可能なユーザー・ゲートウェイ・端末の範囲を決める関係。 |

## 機能・運用

| 用語 | 定義 |
| --- | --- |
| お知らせ | 一般ユーザーが作成し、端末へ表示する情報コンテンツ。 |
| 配信ジョブ | お知らせをどの端末にいつ反映するかを表す処理単位。 |
| 配信ワーカー | 配信ジョブを処理し、配信対象端末の解決・配信実行・結果反映を行うコンポーネント（`remoboard-cloud-delivery`）。 |
| 配信履歴 | 配信ジョブの実行結果（成功・失敗、時刻など）を記録した履歴。 |
| 即時配信 | お知らせ設定後、すぐに端末へ反映する配信方式。 |
| 予約配信 | 指定時刻にお知らせを反映する配信方式。初期リリースでは後回し対象。 |
| コールバック | ゲートウェイからクラウドAPIへ、配信結果などを通知する呼び出し。 |

## セキュリティ・認証

| 用語 | 定義 |
| --- | --- |
| 認証 | 利用者が正当なユーザーであることを確認する仕組み。 |
| 認可 | 認証後のユーザーに対し、ロールと所有関係に基づいて操作可能範囲を制御する仕組み。 |
| ゲートウェイトークン | クラウドとゲートウェイ間認証に使う、ゲートウェイごとに異なるトークン。 |
| ペアリングコード | 端末とゲートウェイを紐づけるためのコード。使い捨て・短時間有効を想定。 |
| QRコード | 端末とゲートウェイの紐づけに利用するコード形式。 |
| 監査ログ | 重要操作や設定変更を追跡するための操作履歴ログ。 |
| 秘密情報 | パスワード・トークン・鍵など、環境変数や安全なシークレット管理で扱うべき情報。 |
