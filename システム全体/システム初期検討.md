# 遠隔お知らせ掲示板 初期検討

## 目的
- GitHubでの開発手法を習得するために、クラウドシステムを開発する
- 遠隔お知らせ掲示板の開発を通じて、クラウドシステムの設計・実装・運用のスキルを向上させる
- ユーザーが遠隔から自宅端末へお知らせを表示できる仕組みを題材に、Web・API・ワーカー・データベースを含むシステム全体を検討する

## 検討対象スコープ

- 本ドキュメントは、初期検討段階におけるシステムの全体像、前提条件、基本機能、セキュリティ方針を整理することを目的とする
- 初期段階では、主にクラウド側システムを開発対象とする
- ゲートウェイおよび端末は、実機ではなく模擬ゲートウェイ・模擬端末での動作確認を前提とする
- 冗長化や大規模運用は初期リリースの対象外とし、まずは単一構成で成立することを重視する

## アーキテクチャ

### クラウド側のシステム

- ユーザー操作用Webサーバー `remoboard-cloud-web-user`（React / Node）
- ユーザー操作用REST APIサーバー `remoboard-cloud-api-user`（FastAPI / Python）
- 管理者用Webサーバー `remoboard-cloud-web-admin`（React / Node）
- 管理者用REST APIサーバー `remoboard-cloud-api-admin`（FastAPI / Python）
- ゲートウェイ向け配信制御ワーカー `remoboard-cloud-delivery`（Python）
- データベース `PostgreSQL`

### 初期段階での責務イメージ

- Webサーバーは、ユーザーおよび管理者の画面を提供する
- APIサーバーは、認証、端末管理、配信設定、管理機能を提供する
- 配信制御ワーカーは、配信対象端末の解決、配信ジョブの実行、配信結果の反映を行う
- データベースは、ユーザー、ゲートウェイ、端末、お知らせ、配信履歴、監査ログを保持する

### ユーザー自宅のゲートウェイ

- ユーザー自宅のゲートウェイに相当する `remoboard-gateway-api`（Express / Node）
- 初期段階では、模擬ゲートウェイとしてクラウドからの指示受信、端末一覧返却、端末への表示反映を確認できればよい

## 開発方法

- GitHub を使用する
- GitHub Copilot を使用する
- Pythonは、PyCharmを使用する
- Nodeは、WebStormを使用する
- GitHubのリポジトリは、以下のように構成する
  - remoboard-docs: ドキュメント
  - remoboard-cloud-web-user: クラウド側のユーザー操作用webサーバー
  - remoboard-cloud-api-user: クラウド側のユーザー操作用REST APIサーバー
  - remoboard-cloud-web-admin: クラウド側の管理者用webサーバー
  - remoboard-cloud-api-admin: クラウド側の管理者用REST APIサーバー
  - remoboard-cloud-delivery: クラウド側のユーザー自宅ゲートウェイ制御ワーカー
  - remoboard-gateway-api: ユーザー自宅ゲートウェイAPIサーバー, Express Node
- Linux上での実行、または Docker コンテナ上での実行を想定する
- GitHub Actions で CI を行う
  - マークダウンファイルのチェック
  - Pythonのコードのチェック
  - Nodeのコードのチェック
  - Pythonのコードのテスト
  - Nodeのコードのテスト
  - コンテナのビルド
  - コンテナイメージのプッシュ
- 初期段階では Kubernetes は使用しない
- 初期段階では 冗長化は行わない
- 認証情報、固有情報は、ソースファイルやコンテナイメージに含めない

## 役割

- `admin管理者`
  - システム全体を管理する役割
  - システム設定、全体監視、権限管理を行う
  - `remoboard-cloud-web-admin` を使用する
- `一般管理者`
  - `adminユーザー` を管理する役割
  - `adminユーザー` の作成・停止・所属管理を行う
  - 一般管理者の情報は、`admin管理者` のみが閲覧できる
  - `remoboard-cloud-web-user` を使用する
- `adminユーザー`
  - 担当するゲートウェイ群および一般ユーザーを管理する役割
  - `remoboard-cloud-web-admin` を使用する
- `一般ユーザー`
  - 実際に端末へお知らせを配信する役割
  - `remoboard-cloud-web-user` を使用する

## 前提条件

- 本システムは、クラウドシステムを主開発対象とし、ゲートウェイ以降は模擬ゲートウェイおよび模擬端末での動作を想定する
- 構成の基本単位は「ゲートウェイ1台 : 端末N台」とする
- 端末は同時に1つのゲートウェイにのみ紐づけられる
- ゲートウェイは、1人のadminユーザーの管理下に置かれる
- 一般ユーザーは、自分に利用許可されたゲートウェイ配下の端末のみ利用できる

- adminユーザーが、ゲートウェイの登録・削除・状態確認を行う
- adminユーザーが、一般ユーザーへゲートウェイの利用権を設定する
- クラウドからゲートウェイへの認証は、ゲートウェイごとに異なるトークンで行う
- ゲートウェイ削除時は、配下端末との紐づけを解除し、当該端末への配信を停止する

- 一般ユーザーが、端末の追加・削除を行う
- 一般ユーザーが、端末とゲートウェイの紐づけ・紐づけ解除を行う
- 端末とゲートウェイの紐づけは、ペアリングコードまたはQRコードによる明示的な登録操作で行う
- 一般ユーザーが、端末を指定して表示内容を設定する
- 一般ユーザーが、自分の端末の状態を確認する

- admin管理者が、システム全体の設定、監視、権限管理を行う
- admin管理者は、全ユーザーの情報を閲覧できる
- 一般管理者の情報は、admin管理者だけが閲覧できる
- 一般管理者が、adminユーザーの作成・停止・所属管理を行う
- 一般管理者は、adminユーザーの情報を閲覧できる
- adminユーザーは、自分が管理対象として割り当てられた一般ユーザーだけを管理できる
- adminユーザーは、自分が管理するゲートウェイだけを管理できる
- 一般ユーザーは、自分に紐づく端末だけを管理できる

## 基本機能

### ユーザー目線

- ログインして、自分に利用許可されたゲートウェイ・端末を確認できる
- 端末を追加できる（ペアリングコードまたはQRコードで登録）
- 端末を削除できる（削除後は当該端末への配信を停止する）
- 端末とゲートウェイの紐づけ・紐づけ解除・再紐づけができる
- 端末一覧で、接続状態・最終接続時刻・現在の紐づけ先ゲートウェイを確認できる
- お知らせを新規作成し、表示情報として登録できる
- 表示情報を設定したとき、ただちに端末へ反映させる方法と、予約した時刻に反映させる方法を選択できる
- お知らせの編集・削除ができる（配信前の修正、配信取り下げを含む）
- 配信先端末を選択して、どの端末に表示するか指定できる
- 配信履歴や配信結果（成功・失敗）を確認できる
- 自宅端末の現在表示内容と最終更新時刻を確認できる

### 管理者目線

- ユーザーアカウントの作成・停止・削除ができる
- adminユーザーと一般ユーザーの所属関係・利用権限を管理できる
- ユーザーごとの契約状態や利用枠（端末数、配信数）を管理できる
- ゲートウェイの登録・削除・状態確認ができる
- ゲートウェイごとの端末紐づけ状況、オンライン状態、最終接続時刻を監視できる
- 端末の登録状況や利用ユーザーとの対応関係を確認できる
- システム全体の配信ジョブ状況を監視し、失敗時に再実行できる
- 監査ログ（操作履歴）を参照し、不正利用や設定ミスを追跡できる
- メンテナンス告知や障害情報をユーザーへ通知できる

## 初期リリースで優先する範囲

- ユーザーのログイン
- ゲートウェイ一覧・端末一覧の表示
- 端末の追加、削除、紐づけ
- お知らせの即時配信
- 配信結果の確認
- 管理者によるユーザー管理、ゲートウェイ管理
- ゲートウェイおよび端末の状態確認

- お知らせの予約配信
- 配信失敗時の高度な再送制御
- 契約プランや利用量課金の詳細管理
- 高度な分析画面や統計ダッシュボード
- 障害通知の自動化や外部通知連携

## セキュリティ

- ユーザー認証は、メールアドレス・パスワードによるログイン方式を採用する
- パスワードはハッシュ化して保存する（bcryptなどを使用）
- API通信はHTTPSを使用し、通信内容を暗号化する
- ゲートウェイとクラウド間の通信もHTTPSを使用する
- ゲートウェイ認証は、ゲートウェイごとに異なるトークンを使用する
- 認可は、役割と所有関係に基づいて制御する
- ペアリングコードは、使い捨てかつ短時間有効とする
- 監査ログを保持し、重要操作を追跡できるようにする
- 秘密情報は環境変数や安全なシークレット管理機構で扱う
