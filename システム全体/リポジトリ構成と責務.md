# リポジトリ構成と責務

## 目的

- 本ドキュメントは、遠隔お知らせ掲示板システムのリポジトリ構成と責務分離を一覧化する
- 初期検討時点の合意内容を記載し、詳細仕様は各リポジトリで管理する

## リポジトリ一覧

| リポジトリ | 主な責務 | 主な利用者/実行主体 | 主な技術スタック |
| --- | --- | --- | --- |
| `remoboard-docs` | 全体方針、設計、用語、運用方針などのドキュメント管理 | 全開発メンバー | Markdown |
| `remoboard-cloud-web-user` | 一般ユーザー向け画面の提供（ログイン、端末管理、お知らせ操作、配信結果確認） | 一般ユーザー | React / Node |
| `remoboard-cloud-api-user` | 一般ユーザー向け業務APIの提供（認証、端末管理、お知らせ管理、配信要求） | `remoboard-cloud-web-user` / 外部クライアント | FastAPI / Python |
| `remoboard-cloud-web-admin` | 管理者向け画面の提供（ユーザー管理、ゲートウェイ管理、監査確認） | admin管理者 / 一般管理者 / adminユーザー | React / Node |
| `remoboard-cloud-api-admin` | 管理者向け業務APIの提供（管理者認証、ユーザー管理、ゲートウェイ管理、監査ログ参照） | `remoboard-cloud-web-admin` / 管理系クライアント | FastAPI / Python |
| `remoboard-cloud-delivery` | 配信ジョブ処理（対象端末解決、ゲートウェイ配信要求、配信結果反映、再実行制御） | ワーカー（バッチ/非同期処理） | Python |
| `remoboard-gateway-api` | 自宅ゲートウェイ機能（クラウド指示受信、端末一覧返却、端末表示反映） | ユーザー自宅ゲートウェイ | Express / Node |

## 責務分離の方針

- 画面（Web）と業務ロジック（API）を分離し、ユーザー系と管理者系も分離する
- 配信処理はワーカーとして分離し、API応答性能と配信処理を切り離す
- ゲートウェイ側APIはクラウド側と分離し、宅内制御の責務を独立させる
- 共通の設計判断や運用方針は `remoboard-docs` で一元管理する

## 初期段階の主な連携

- `remoboard-cloud-web-user` -> `remoboard-cloud-api-user`
- `remoboard-cloud-web-admin` -> `remoboard-cloud-api-admin`
- `remoboard-cloud-api-user` -> `remoboard-cloud-delivery`（配信ジョブ投入）
- `remoboard-cloud-delivery` -> `remoboard-gateway-api`（配信要求）
- 各クラウド側コンポーネント -> PostgreSQL（ユーザー、端末、配信、監査など）
